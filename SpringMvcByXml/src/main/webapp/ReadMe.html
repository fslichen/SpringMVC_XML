Spring MVC and Spring

Spring MVC and spring are independent, which means that you are allowed to use spring MVC without using spring. As a matter of fact, I've deleted all the spring configurations. There can be no listener for spring, there can be no applicationContext.xml

Spring MVC Maven

In maven defintions, you can delete all the spring related maven dependencies, such as the spring-core, spring-context, or spirng-web. And spring-webmvc is all you need.

web.xml Configurations

In web.xml, all you need is just a servlet definition, like what you did in the basic servlet definition, you can do the same thing for spring-mvc.

spring-mvc.xml Configurations

The configurations for spring-mvc.xml is much like applicationContext.xml. You can define spring beans inside, and use component-scan. Please don't forget to put component-scan in your configuration file, otherwise spring-mvc is not going to detect the @RestController annotation inside the controller, therefore you will encounter lots of 404s. And MappingJackson2HttpMessageConverter can transfer a json string into a POJO by using @RequestBody inside the controller.
